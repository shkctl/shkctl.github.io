# 编辑docker-compose.yml

我们到自己的服务器中创建创建一个docker-compose.yml，文件内容如下，读者可以基于笔者的注释进行进一步的修改,这里补充说一句docker-compose.yml对格式的要求很高，笔者在前期配置时遇到各种should be array的错误，大部分增加空格都能解决，读者配置的时候需要留意一下。

# docker-compose版本3的规范
version: "3"
services:
#  mysql的配置
  mysql:
    container_name: mysql
#    选择MySQL 5.7的镜像
    image: mysql:5.7
    environment:
#      mysql密码
    - MYSQL_ROOT_PASSWORD=xxxxxx
    volumes:
#      设置容器内部地址和外部地址的映射
    - /app/cloud/mysql/data:/var/lib/mysql
    ports:
#      启动端口号
    - "3306:3306"
    restart: always

  nacos:
    image: nacos/nacos-server:1.1.4
    container_name: nacos
    environment:
      配置nacos持久化为MySQL并指定MySQL配置信息
    - PREFER_HOST_MODE=hostname
    - MODE=standalone
#    使用单数据库模式
    - MYSQL_DATABASE_NUM=1
    - SPRING_DATASOURCE_PLATFORM=mysql
    - MYSQL_MASTER_SERVICE_HOST=mysql
#    数据库名称
    - MYSQL_MASTER_SERVICE_DB_NAME=nacos_config
    - MYSQL_MASTER_SERVICE_PORT=3306
    - MYSQL_MASTER_SERVICE_USER=xxx
    - MYSQL_MASTER_SERVICE_PASSWORD=xxxx
    volumes:
    - /app/cloud/nacos/logs:/home/nacos/logs
    ports:
    - "8848:8848"
    depends_on:
    - mysql
    restart: always


  sentinel:
    image: bladex/sentinel-dashboard:latest
    container_name : sentinel
    ports:
    - "8858:8858"
    environment:
#      设置sentinel账户密码信息
      sentinel.dashboard.auth.username : sentinel
      sentinel.dashboard.auth.password: xxxx
    restart : always

docker-compose up -d
1

此时我们会发现nacos会启动失败，原因很简单，我们的MySQL不仅账号密码不正确，而且nacos_config数据库也不存在，这一点读者可以到服务器本地目录的/app/cloud/nacos/logs的start.out知晓。

# 修改MySQL配置信息并创建nacos数据库

所以我们必须手动修改一下数据库信息并且创建nacos数据库。首先我们使用docker ps查看当前MySQL容器的id。

以笔者为例，看到MySQL的容器id为2099e8aac607 ，于是键入下面这段命令进入容器，进行数据库信息修改

 docker exec -it 2099e8aac607 bash
1

键入mysql -u root -p输入密码(12345)，然后一条一条执行下面的命令完成密码修改

# 使用MySQL
use mysql;

select host,user from user;

ALTER USER 'root'@'%' IDENTIFIED WITH mysql_native_password BY '密码';
# 刷新
flush privileges;
