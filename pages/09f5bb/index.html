如何阅读一本书：技术书籍的读书笔记方法论

# 写在文章开头

很多人都有这样的经历：读完一本书，合上封面，却想不起书里到底说了什么。划了一堆重点，回头翻看全是别人的话，没有一句是自己的理解。这不是记忆力的问题，而是阅读方式的问题。

《如何阅读一本书》作者莫提默·J·艾德勒（Mortimer J. Adler）指出，阅读分为三个目的：

  1. 娱乐消遣
  2. 获取资讯
  3. 增进理解力

只有第三种——以增进理解力为目标的主动阅读——才能真正让知识内化。而主动阅读的核心，就是与作者对话：质疑、回应、辩论、标记。他在1941年发表于《星期六文学评论》的文章《如何标记一本书》中，进一步提出了一个具体的实践方法：在书上做标记，把阅读从被动接收变成主动思考的过程。

本文基于 Adler 的原文，提炼其核心观点，并以阅读《Effective Java》为例，展开说明笔者在阅读技术书籍时的标记和笔记技巧，希望对你有帮助。

# 正确的读书观

Adler 指出，拥有一本书有两种方式：第一种是财产所有权——花钱买下它，就像买衣服和家具一样，但这只是占有的前奏；第二种是完全拥有——把书变成自己的一部分，而最好的方式就是在书上写作。正如你买了一块牛排，把它从肉铺的冰箱转移到自己的冰箱里，但直到你把它消化吸收进血液中，你才算真正拥有了这块牛排。好书也是如此——买回来放在书架上不算拥有，只有你主动阅读、思考消化、做笔记并用自己的话输出后，才算真正掌握了书中的知识。

很多人对纸张、装帧和印刷怀有盲目敬畏，觉得在书上写字是对书的亵渎。但 Adler 认为这恰恰是对书的错误尊重——尊重的是印刷工艺而非作者的思想。拥有一个精美的书架并不能证明主人的思想因书而丰富，只能证明他足够有钱买下它们。Adler 将藏书人分为三种：

  1. 第一种人拥有所有标准套装和畅销书——未读、未碰。他们拥有的只是木浆和墨水，不是书
  2. 第二种人拥有很多书——少数读完，多数翻过，但全部崭新如初。他们想让书成为自己的，却被对物理外观的虚假尊重所束缚
  3. 第三种人拥有少数或很多书——每一本都折角、破旧、因反复使用而松散，从头到尾写满标记。这个人真正拥有书

第三种正是本文所谈论的重点——在书上做标记的人。书的破旧程度，恰恰是你与它对话的深度证明。

有人会问：保持一本精美印刷、优雅装帧的书完好无损，难道不是错误的尊重吗？当然不是——Adler 不会在《失乐园》初版上涂鸦，就像不会把蜡笔交给婴儿在伦勃朗原作上画画。画作和雕塑的灵魂与身体不可分离，珍稀版本的美也是如此。

但书不同，书更像乐谱而非画作。伟大的指挥家不会把交响乐和印刷乐谱混为一谈——托斯卡尼尼对勃拉姆斯充满敬意，但他的 C 小调交响乐乐谱上密密麻麻写满标记，除了他本人没人能看懂。伟大指挥家之所以反复在乐谱上做标注，正是你应该在书上做标记的原因。如果对精美装帧的敬畏妨碍了你，那就买个廉价版本，把你的敬意献给作者。

# 阅读是一场对话

阅读一本书应该是你与作者之间的对话。

理解是双向的——学习不是做一个空容器。学习者必须质疑自己、质疑老师，甚至在理解老师所说的内容后与老师辩论。标记书本质上就是表达你与作者之间的分歧或共识。

具体来说，这种对话体现在三个层面：

  1. 质疑 — 读到某个结论时问自己"真的是这样吗？"例如读到"复合优先于继承"时，你可以追问"那 Java 的类库本身不也在用继承吗？AbstractList、AbstractSet 是不是反例？"这种质疑不是抬杠，而是迫使你理解作者结论的适用范围
  2. 回应 — 用自己的话重新解释作者的观点，写下你同意或不同意的理由。例如读完第18条后写下"有点类似于装饰者模式的思想"，这就是你的回应——你在用自己的知识体系回应作者的论点
  3. 联想 — 把作者的观点和你已有的知识建立连接。例如读到"避免创建不必要的对象"时，联想到项目中 String 拼接导致的性能问题，在页边记录这个关联。这种联想让新知识锚定在旧知识上，不容易遗忘

# 为何需要书本笔记

Adler 说：在书上做标记不是破坏，而是热爱。 阅读不仅要"读字里行间"（read between the lines），更要"写字里行间"（write between the lines）。不做标记的阅读不是最高效的阅读，研究从以下几个角度解释了在书本上做笔记对学习的具体好处：

# 促进深度理解

单纯阅读是被动的信息接收，大脑容易进入"自动驾驶"模式——眼睛扫过文字，信息短暂停留在工作记忆中，很快就被新内容覆盖。而做笔记迫使你停下来处理信息，大脑必须回答三个问题：

  1. 这段话在说什么？
  2. 它和前面的论述有什么关系？
  3. 我同意吗？

每回答一次，都会进行一次主动思考也就是信息处理的过程。大脑就对这个知识点做了一次深层加工。Adler 也说过：主动阅读本质上是思考过程，而思考倾向于用语言来表达——做了标记的书，通常是经过深度思考的书。

芝加哥大学校长 Hutchins 总是带着铅笔读书，当发现自己只在页边画涂鸦时，就知道太累了不该再读。做标记能让你保持清醒、全神贯注，而不是仅仅保持意识。

以读《Effective Java》第18条"复合优先于继承"为例。作者先指出继承的问题：假设你想记录一个 HashSet 添加了多少个元素，于是写了一个 InstrumentedHashSet 继承 HashSet：

public class InstrumentedHashSet<E> extends HashSet<E> {
    private int addCount = 0;

    @Override public boolean add(E e) {
        addCount++;
        return super.add(e);
    }

    @Override public boolean addAll(Collection<? extends E> c) {
        addCount += c.size();       // 期望加 3
        return super.addAll(c);     // 但 HashSet.addAll() 内部会逐个调用 add()，又加了 3
    }

    public int getAddCount() { return addCount; }
}

看起来没问题，但调用 addAll(Arrays.asList("Snap", "Crackle", "Pop")) 时，getAddCount() 返回 6 而不是 3——因为 HashSet 的 addAll() 内部实现是逐个调用 add()，计数被重复执行。这就是继承破坏封装性的典型例子：子类依赖了超类的实现细节，而超类没有义务保持这种细节不变。

作者的解决方案是复合加转发——不继承 HashSet，而是持有一个 Set 的引用，把所有方法调用转发给它：

public class ForwardingSet<E> implements Set<E> {
    private final Set<E> s;    // 持有引用，不继承

    public ForwardingSet(Set<E> s) { this.s = s; }

    @Override public boolean add(E e)             { return s.add(e); }     // 转发
    @Override public boolean addAll(Collection<? extends E> c) { return s.addAll(c); }  // 转发
    // ... 其他 Set 方法同理转发
}

先完整读完这一条，理解作者的完整论证后，开始做笔记：

  1. 标记核心结论 — 在"继承会破坏封装性"下划线，标注这一条的核心论点
  2. 梳理论证链条 — 在页边用数字标注推导顺序：①子类依赖超类实现细节 → ②超类改动导致子类行为异常 → ③复合加转发是更安全的替代方案
  3. 写下个人理解 — 在页边写下自己的理解："有点类似于装饰者模式的思想"

这种"先读完再标记"的方式，比边读边停更好，因为你掌握了完整的论证脉络后，才知道哪些是铺垫、哪些是结论、哪些值得标记。美国全国英语教师委员会（NCTE）发表的教学研究（Porter-O'Donnell, 2004）指出，批注是一种强大的阅读策略，能帮助读者理解困难材料并保持主动阅读。

# 强化记忆保持

被动阅读时，信息只经过大脑的工作记忆——一个容量有限、衰减极快的临时缓冲区。你以为"读过了就是记住了"，但其实十分钟后大部分内容已经模糊。手写笔记能把信息从工作记忆推进到长期记忆，原因是它迫使大脑做了一次"翻译"：从作者的表述，翻译成你自己的语言。这个过程认知心理学称之为"生成效应"（Generation Effect，Slamecka & Graf, 1978）——自己产出的信息，比被动接收的信息记忆效果显著更好。

挪威科技大学的脑电实验（Van der Weel & Van der Meer, 2023）从神经科学层面印证了这一点：手写时大脑多个区域同时被激活，脑区连接模式显著强于打字。普林斯顿大学和加州大学洛杉矶分校的联合实验（Mueller & Oppenheimer, 2014）也证实：手写笔记组在概念理解题上显著优于打字组。

还是以《Effective Java》为例。读完第18条后，你在页边写下：

总结：①继承让子类依赖超类实现细节 → ②超类内部改动会导致子类行为异常 → ③用复合（持有引用）加转发（委托调用）替代继承

这段总结不是抄书，而是你用自己的话提炼了三个关键步骤。写的过程中，大脑必须回答：

  1. 核心论点是什么？
  2. 每个论点的原因？
  3. 代码示例片段与最终输出

每回答一次，记忆就更深一层。Adler 也指出：亲手书写能让文字和句子更清晰地呈现在脑海中，并在记忆中保存得更久——即使写在草稿纸上然后扔掉，你对书的理解也会更牢固。

# 构建个人知识体系

批注不仅是记录作者说了什么，更是将外部知识内化为个人认知结构的过程。当你在页边写下自己的理解、质疑或与项目经验的关联时，你正在把作者的知识和自己的知识网络建立连接。这种连接越密集，知识越不容易遗忘，也越容易被灵活调用。Adler 说："做了标记的书，通常是经过深度思考的书。"

# 提高复习效率

标记过的书自带导航系统——下划线标注核心段落，圈注标记关键术语，页边数字梳理论证链条。复习时不需要重读全书，只需扫一遍标记就能快速回忆起完整的知识框架。纸质书的物理特征（折角位置、批注方位、标记颜色）还会形成空间记忆线索，你能回忆起"某个知识点在左页第三段"，正是因为这些空间锚点。

# 详解读书标记技巧

Adler 原文提出了七种标记方法，笔者在此基础上按用途简化为以下体系，以阅读《Effective Java》为例逐一说明。

# 下划线：标注核心论述段落

遇到作者给出核心结论或关键论证的段落时，用下划线划出。这些段落通常是每节的核心内容，下划线帮助你在回顾时快速定位作者最重要的论述。例如读到"创建和销毁对象"章节时，"用静态工厂方法代替构造器"这一结论及其三个优势（有名称、不必每次创建新对象、可以返回子类型对象）都应该划线标注。

# ○ 圈注：标注术语关键词

遇到反复出现的核心术语时，用圆圈圈出。这些术语是理解全书的钥匙，圈出来后翻阅时可以快速定位所有相关讨论。例如《Effective Java》中的"快速失败（fail-fast）"、"防御性拷贝（defensive copy）"、"线程安全（thread-safe）"等术语，每出现一次就圈一次，后续复习时只需扫一眼圈注就能找到所有涉及该概念的段落。

# ①②③ 观点梳理：梳理论证链条中观点的顺序

当作者在一个段落内分步骤展开论证时，在页边用数字标注每个观点的顺序，帮你理清"前提→推导→结论"的论证链条。例如"避免创建不必要的对象"一节中，作者依次论证了三个层次：字符串拼接的性能问题 → 重用不可变对象 → 重用不会被修改的可变对象。在页边标注 ①②③ 后，论证结构一目了然，复习时不需要重读全文就能回忆起完整的推理过程。

# ★ 星号：标记全书最重要的金句

星号是最珍贵的标记，它标记的是重要的理论或者出色的代码段。它标记的不是"重要"，而是"最重要"——那些你未来可能直接引用、或者可直接复用的编码思想。例如：

  1. 设计原则，如"复合优先于继承"
  2. 核心代码片段，如某种线程安全的单例实现、某种并发工具的经典用法
  3. 性能优化建议，如"避免在循环中创建对象"

# # 批注：对某个知识点的个人理解、评论、解析

读到某个知识点时，如果脑海中产生了自己的理解、质疑或联想，立刻在旁边用 # 开头写下来。批注的内容可以是：用自己的话重新解释这段内容、记录与项目经验的关联、提出疑问、补充自己的见解。例如读到第18条"复合优先于继承"时，可以写下："有点类似于装饰者模式的思想"。批注是你与作者对话的痕迹，也是日后复习时最有价值的部分——因为它是你自己的思考，而非作者的原话。

# 总结：读完一个完整知识点后，用自己的话提取要点

每读完一个完整知识点或一小节，在段落末尾的空白处写"总结："，然后分条列出核心要点。总结不是摘抄作者的原话，而是用自己的语言重新组织。如果总结不出来，说明这个知识点还没有真正理解，需要回去重读。例如读完"异常"这一章后，可以写下：

  1. 异常仅用于异常场景，不要用异常控制程序流程
  2. 受检异常用于可恢复场景，非受检异常用于编程错误
  3. 异常信息应包含失败时的所有关键参数，便于定位问题
  4. 努力保证失败原子性——方法调用失败后对象应保持调用前的状态

# 关于阅读速度的误解

标记书会放慢阅读速度吗？是的，而这正是应该这样做的原因之一。Adler 指出：不存在所谓"正确的"智能阅读速度。有些东西应该快速轻松地读，有些则应该缓慢甚至费力地读。阅读中智力的标志，是根据内容的价值以不同方式阅读不同内容的能力。

读技术书籍更是如此。你可以在一个下午翻完一本框架入门书，因为大部分内容是 API 用法和示例代码，理解门槛低，快速浏览即可。但读《Effective Java》第18条"复合优先于继承"时，你可能要花半小时——先读懂 InstrumentedHashSet 的 bug，再理解 ForwardingSet 的转发机制，最后还要停下来关联思考"这和装饰者模式有什么关系"。这种新旧知识的关联就是精细加工——大脑中的知识不是孤立存储的，而是通过连接形成网络，新旧知识互相印证，连接越多，记忆越稳固。这半小时不是慢，而是你在思考。好书的关键不在于你读完了多少本，而在于有多少本穿透了你——有多少本你真正变成了自己的。就像少数知己胜过上千泛泛之交。

# 小结

本文基于 Adler 的《如何标记一本书》，结合笔者的实践经验，以阅读《Effective Java》为例展开说明：

  1. 正确的读书观 — 买书只是财产所有权，在书上做标记才是完全拥有。藏书人分三种，真正拥有书的人是那些把书折角、破旧、写满标记的人。如果对精美装帧的敬畏妨碍了你，就买个廉价版本，把敬意献给作者
  2. 阅读是一场对话 — 主动阅读不是被动接收，而是与作者对话：质疑结论的适用范围、用自己的话回应观点、把新知识和已有知识建立联想
  3. 为何需要书本笔记 — 做笔记促进深度理解、强化记忆保持、构建个人知识体系、提高复习效率，科学研究从多个层面印证了这一点
  4. 详解读书标记技巧 — 按用途简化为六种：下划线标注核心论述、圈注标记术语、数字梳理论证链条、星号标记金句（如设计原则、核心代码片段、性能优化建议）、批注写下个人理解、读完后用自己的话总结
  5. 关于阅读速度 — 不存在正确的智能阅读速度，好书的关键不在于读完了多少本，而在于有多少本穿透了你
